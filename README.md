# پلتفرم نوبت‌دهی — Booking Platform

پلتفرم رزرو و نوبت‌دهی آنلاین با فرانت‌اند و بک‌اند مجزا، طراحی‌شده با تمرکز بر امنیت، صحت داده و مقیاس‌پذیری.

## وضعیت فعلی: فاز ۲ تکمیل شد — پروفایل و ثبت کسب‌وکار

**فاز ۰** (پایه‌ریزی): schema کامل دیتابیس، مستندات معماری/API/design-system، اسکلت مانو-ریپو، Docker، CI.

**فاز ۱**: احراز هویت کامل (OTP، JWT+refresh rotation، مدیریت نشست، RBAC سراسری).

**فاز ۲** (این نسخه) شامل:
- ویزارد ثبت کسب‌وکار: ایجاد → مدارک KYC → صف بررسی ادمین → تایید/رد/تعلیق (با audit log)
- مدیریت شعبه، خدمات (با اختصاص به کارمند مشخص)، کارکنان (دعوت/غیرفعال‌سازی با محافظت OWNER)
- ساعات کاری و تعطیلات (برای شعبه یا یک کارمند خاص، با اعتبارسنجی تداخل زمانی)
- سیاست لغو قابل‌تنظیم به ازای هر کسب‌وکار
- جستجو و مشاهده‌ی عمومی کسب‌وکارهای تایید‌شده (`/discover/businesses`)
- `BusinessRoleGuard`: کنترل دسترسی سطح کسب‌وکار (OWNER/MANAGER/STAFF)، مستقل از نقش سراسری، با bypass برای ادمین
- ۴۹ تست واحد واقعی (۳۰ تست جدید این فاز، شامل ۷ تست فقط برای منطق امنیتی guard)

### حالت موقت OTP قبل از اتصال پیامک واقعی
تا وقتی یک provider واقعی پیامک (کاوه‌نگار و مشابه) وصل نشده، با `OTP_DEBUG_EXPOSE=true` در `.env`، کد OTP مستقیم در پاسخ API (فیلد `debugCode`) برمی‌گرده تا فرانت بتونه همون صفحه نشونش بده. این حالت **صرف‌نظر از مقدار env، در `NODE_ENV=production` همیشه غیرفعاله** — یک تنظیم فراموش‌شده نمی‌تونه در محیط واقعی کد رو لو بده. قبل از انتشار نهایی، باید هم این flag رو false کنید هم یک `SmsProvider` واقعی (پیاده‌سازی interface موجود در `common/sms/`) جایگزین `ConsoleSmsProvider` بشه.

منطق دامنه‌ی بعدی (رزرو، پرداخت) در فازهای ۳ به بعد اضافه می‌شه.

## راه‌اندازی محیط توسعه

```bash
# ۱. سرویس‌های زیرساختی (Postgres + Redis)
docker compose up -d

# ۲. نصب وابستگی‌ها (در ریشه‌ی مانو-ریپو)
npm install

# ۳. کپی فایل env
cp .env.example .env
# مقادیر واقعی (secrets) رو داخل .env جایگزین کنید — هرگز commit نشه

# ۴. اجرای migration اولیه
npm run prisma:migrate

# ۵. اجرای backend
npm run dev:backend

# ۶. اجرای frontend
npm run dev:frontend
```

## ساختار پروژه

```
/apps/backend    → NestJS API
/apps/frontend   → Next.js
/packages/shared → تایپ‌ها/enumهای مشترک
/prisma          → schema دیتابیس
/docs            → مستندات معماری و API
```

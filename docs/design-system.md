# Design System (نسخه‌ی پایه)

هدف: یک زبان طراحی یکپارچه بین صفحات کاربر/کسب‌وکار/ادمین، قابل‌پیاده‌سازی به‌صورت CSS Variables + Tailwind config در فرانت‌اند Next.js.

## رنگ‌ها (Design Tokens)

| نام Token | مقدار | کاربرد |
|---|---|---|
| `--color-primary` | #2563EB | دکمه‌های اصلی، لینک‌ها |
| `--color-primary-dark` | #1E40AF | hover حالت اصلی |
| `--color-success` | #16A34A | تایید رزرو، پرداخت موفق |
| `--color-warning` | #D97706 | وضعیت PENDING، هشدار |
| `--color-danger` | #DC2626 | لغو، خطا، حذف |
| `--color-neutral-900` | #111827 | متن اصلی |
| `--color-neutral-500` | #6B7280 | متن ثانویه |
| `--color-neutral-100` | #F3F4F6 | پس‌زمینه کارت‌ها |
| `--color-surface` | #FFFFFF | پس‌زمینه اصلی |

## تایپوگرافی
- فونت فارسی: Vazirmatn (پشتیبانی کامل RTL و اعداد فارسی)
- مقیاس: `text-xs` تا `text-3xl` بر پایه‌ی سیستم Tailwind پیش‌فرض، بدون سفارشی‌سازی غیرضروری

## فاصله‌گذاری و شبکه
- واحد پایه: 4px (مقیاس Tailwind استاندارد: 1=4px, 2=8px, ...)
- Container عرض حداکثر: 1280px با padding افقی 16px در موبایل

## اصول رابط کاربری
- **RTL native** — همه‌ی کامپوننت‌ها از ابتدا برای راست‌به‌چپ طراحی میشن، نه mirror بعدی.
- **حالت لودینگ/خالی/خطا برای هر کامپوننت داده‌محور اجباریه** — این‌ها بخشی از تعریف "کامل بودن" یک کامپوننته، نه یک افزوده‌ی بعدی.
- **دسترسی‌پذیری (a11y)**: کنتراست رنگ حداقل AA، تمام فرم‌ها label معتبر دارن، ناوبری با کیبورد پشتیبانی میشه.
- **کامپوننت‌های پایه** که باید اول ساخته بشن: Button, Input, Select, Modal, Toast, Card, Badge (وضعیت رزرو/پرداخت), Calendar/DatePicker, DataTable.

## نکته‌ی معماری فرانت
کامپوننت‌های پایه در `apps/frontend/src/components/ui` قرار می‌گیرن و **مستقل از منطق دامنه** (booking/business) نوشته میشن؛ کامپوننت‌های دامنه (`BookingCard`, `BusinessCard`) از این کامپوننت‌های پایه استفاده می‌کنن نه برعکس. این جدا‌سازی همون چیزیه که در نسخه‌ی قبلی پروژه نبود و باعث تکرار کد در ۲۰+ فایل شده بود.
